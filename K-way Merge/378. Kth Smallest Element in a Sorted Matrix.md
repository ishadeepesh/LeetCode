# 🧩 378. Kth Smallest Element in a Sorted Matrix

## 🔴 Problem
Given an `n x n` matrix where each of the rows and columns is sorted in ascending order, return the **kth smallest element** in the matrix.

> Note: It is the kth smallest element in the **sorted order**, not the kth distinct element.

You must solve it with a memory complexity better than `O(n²)`.

---

## ✨ Example

**Input:**
    matrix = [[1,5,9],[10,11,13],[12,13,15]]
    k = 8

**Output:**
    13

**Explanation:**
    Sorted elements:
    [1,5,9,10,11,12,13,13,15]

    8th smallest = 13

---

## 🚀 Approach: Binary Search on Value Space (🔥 Optimal)

### 💡 Idea
- Since the matrix is sorted **row-wise and column-wise**, we can:
  - Apply **binary search on values**, not indices
- Search range:
  - `left = smallest element`
  - `right = largest element`

---

## ⚙️ Key Insight (Important for Interviews)

For a given `mid`, count how many elements are `<= mid` in **O(n)**:

👉 Start from **bottom-left corner**:
- If current value ≤ mid → move right (add all elements above)
- Else → move up

This avoids scanning the entire matrix.

---

## ⏱️ Complexity

- **Time Complexity:** O(n log(max - min))
- **Space Complexity:** O(1)

---

## 💻 Code

```python
class Solution:
    def kthSmallest(self, matrix, k):
        n = len(matrix)

        def countLessEqual(mid):
            row = n - 1
            col = 0
            count = 0

            while row >= 0 and col < n:
                if matrix[row][col] <= mid:
                    count += row + 1
                    col += 1
                else:
                    row -= 1

            return count

        left = matrix[0][0]
        right = matrix[n - 1][n - 1]

        while left < right:
            mid = (left + right) // 2

            if countLessEqual(mid) < k:
                left = mid + 1
            else:
                right = mid

        return left
