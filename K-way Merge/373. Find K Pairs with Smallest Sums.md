# 🧩 373. Find K Pairs with Smallest Sums

## 🔴 Problem
You are given two integer arrays `nums1` and `nums2` sorted in non-decreasing order and an integer `k`.

Define a pair `(u, v)` which consists of one element from the first array and one element from the second array.

Return the `k` pairs `(u1, v1), (u2, v2), ..., (uk, vk)` with the smallest sums.

---

## ✨ Example

**Input:**
    nums1 = [1,7,11]
    nums2 = [2,4,6]
    k = 3

**Output:**
    [[1,2],[1,4],[1,6]]

**Explanation:**
    All possible pairs (sorted by sum):
    [1,2],[1,4],[1,6],[7,2],[7,4],[11,2],[7,6],[11,4],[11,6]

    First 3 pairs:
    [1,2],[1,4],[1,6]

---

## 🚀 Approach: Min Heap (🔥 Optimal & Interview Standard)

### 💡 Idea
- Treat pair sums like a **sorted matrix**
- Instead of generating all pairs, use a **min heap** to always get the next smallest sum

---

## ⚙️ Strategy

1. Initialize heap with:
       (nums1[i] + nums2[0], i, 0)
   for i in range(min(k, len(nums1)))

2. Pop the smallest pair from heap

3. For each popped `(i, j)`:
   - Add pair `[nums1[i], nums2[j]]` to result
   - Push `(i, j+1)` into heap if valid

4. Repeat until k pairs are collected

---

## ⏱️ Complexity

- **Time Complexity:** O(k log k)
- **Space Complexity:** O(k)

---

## 💻 Code

```python
import heapq

class Solution:
    def kSmallestPairs(self, nums1, nums2, k):
        if not nums1 or not nums2:
            return []

        heap = []
        result = []

        # Initialize heap
        for i in range(min(k, len(nums1))):
            heapq.heappush(heap, (nums1[i] + nums2[0], i, 0))

        while heap and k > 0:
            total, i, j = heapq.heappop(heap)
            result.append([nums1[i], nums2[j]])

            if j + 1 < len(nums2):
                heapq.heappush(heap, (nums1[i] + nums2[j + 1], i, j + 1))

            k -= 1

        return result
