# 241. Different Ways to Add Parentheses

# Problem Statement

Given a string `expression` consisting of numbers and operators (`+`, `-`, `*`), return **all possible results** from computing the expression by **adding parentheses in different ways**.

The goal is to evaluate the expression using **all valid parenthesizations**.

You may return the results in **any order**.

### Constraints

* The expression consists of digits and operators `+`, `-`, `*`.
* All intermediate results fit within a **32-bit integer**.
* The number of results will not exceed **10⁴**.

---

# Example

### Example 1

**Input**

```
expression = "2-1-1"
```

**Output**

```
[0,2]
```

### Explanation

Two possible ways to add parentheses:

```
((2 - 1) - 1) = 0
(2 - (1 - 1)) = 2
```

---

### Example 2

**Input**

```
expression = "2*3-4*5"
```

**Output**

```
[-34,-14,-10,-10,10]
```

Possible parenthesizations:

```
(2*(3-(4*5))) = -34
((2*3)-(4*5)) = -14
((2*(3-4))*5) = -10
(2*((3-4)*5)) = -10
(((2*3)-4)*5) = 10
```

---

# Approach

This problem follows a **Divide and Conquer + Recursion** strategy.

### Key Idea

Every operator in the expression can act as a **split point**.

For each operator:

1. Split the expression into **left** and **right** parts.
2. Recursively compute all possible results for both sides.
3. Combine every left result with every right result using the operator.

---

## Steps

1. Iterate through the expression.
2. When an operator is found (`+`, `-`, `*`):

   * Split the expression into:

     * Left substring
     * Right substring
3. Recursively compute results for both substrings.
4. Combine every pair of results using the operator.
5. Store results in a list.

---

# Optimization (Memoization)

The same sub-expressions may appear multiple times.

Example:

```
2*3-4*5
```

Sub-expressions like:

```
3-4
4*5
```

may be recomputed repeatedly.

Using **memoization (`@cache`)** stores results for previously computed expressions and avoids recomputation.

This significantly improves performance.

---

# Code (Python)

```python
from functools import cache
from typing import List

class Solution:
    def diffWaysToCompute(self, expression: str) -> List[int]:

        @cache
        def compute_ways(expr: str) -> List[int]:

            # Base case: expression is a number
            if expr.isdigit():
                return [int(expr)]

            results = []

            for index, char in enumerate(expr):

                if char in "+-*":

                    left_results = compute_ways(expr[:index])
                    right_results = compute_ways(expr[index+1:])

                    for left_value in left_results:
                        for right_value in right_results:

                            if char == "+":
                                results.append(left_value + right_value)

                            elif char == "-":
                                results.append(left_value - right_value)

                            else:
                                results.append(left_value * right_value)

            return results

        return compute_ways(expression)
```

# Dry Run

Expression:

```
2-1-1
```

Possible splits:

### Split at first `-`

```
2 | 1-1
```

Results:

```
left = [2]
right = [0]
```

```
2 - 0 = 2
```

---

### Split at second `-`

```
2-1 | 1
```

Results:

```
left = [1]
right = [1]
```

```
1 - 1 = 0
```

---

Final results:

```
[2, 0]
```

---

# Time Complexity

```
O(N * 2^N)
```

Explanation:

* Each operator can create multiple recursive splits.
* The number of unique expressions grows exponentially.

Memoization reduces repeated computations but the total combinations can still be large.

---

# Space Complexity

```
O(N * 2^N)
```

* Storing results for all subexpressions.
* Recursion stack depth is **O(N)**.

# Takeaways

* The core trick is **splitting the expression at operators**.
* Each operator becomes a **root of a computation tree**.
* Memoization prevents recalculating the same sub-expression multiple times.
