# 95. Unique Binary Search Trees II

## Problem Title

**95. Unique Binary Search Trees II**

---

# Problem Statement

Given an integer `n`, return **all the structurally unique Binary Search Trees (BSTs)** that store values `1` to `n`.

Each tree must satisfy the **BST property**:

* All nodes in the left subtree are **less than the root**
* All nodes in the right subtree are **greater than the root**

Return the result in **any order**.

---

# Example

### Input

```
n = 3
```

### Output

```
[
 [1,null,2,null,3],
 [1,null,3,2],
 [2,1,3],
 [3,1,null,null,2],
 [3,2,null,1]
]
```

### Explanation

All structurally unique BSTs using values `1..3` are generated.

Possible trees:

```
1         1         2         3         3
 \         \       / \       /         /
  2         3     1   3     1         2
   \       /               \        /
    3     2                 2      1
```

Total trees = **5**

---

# Approach

This problem follows a **Divide and Conquer + Recursion + Memoization** strategy.

## Key Idea

Each number from `1..n` can act as the **root of the BST**.

For each root:

1. Recursively generate **all left subtrees** using numbers `< root`.
2. Recursively generate **all right subtrees** using numbers `> root`.
3. Combine every left subtree with every right subtree.

---

## Recursive Structure

For a range `[left, right]`:

```
for root in range(left, right):
    leftTrees = generate(left, root-1)
    rightTrees = generate(root+1, right)
```

Then combine them:

```
for l in leftTrees:
    for r in rightTrees:
        root = TreeNode(root_val)
        root.left = l
        root.right = r
```

---

# Memoization Optimization

Many subtree ranges repeat during recursion.

Example:

```
generate(2,3)
generate(1,2)
```

Using **`@cache`** stores results for each `(left,right)` pair and prevents recomputation.

This greatly improves performance.

---

# Python Code

```python
from functools import cache
from typing import List, Optional

class Solution:
    def generateTrees(self, n: int) -> List[Optional[TreeNode]]:
      @cache 
        def generate(left, right): 
            if left > right: 
                return [None]
            res = [] 
            for i in range(left, right + 1): 
                leftTrees = generate(left, i - 1) 
                rightTrees = generate(i + 1, right) 
                for leftTree in leftTrees: 
                    for rightTree in rightTrees: 
                        root = TreeNode(i, leftTree, rightTree) 
                        res.append(root) 
            return res 
        return generate(1, n)
```

# Dry Run

For `n = 3`

Possible roots:

```
1, 2, 3
```

### Root = 1

```
Left  = []
Right = trees from [2,3]
```

### Root = 2

```
Left  = [1]
Right = [3]
```

### Root = 3

```
Left  = trees from [1,2]
Right = []
```

Combining all possibilities produces **5 unique BSTs**.

---

# Time Complexity

```
O(Cn)
```

Where `Cn` is the **Catalan Number**.

Approximation:

```
O(4^n / n^(3/2))
```

This represents the number of unique BST structures.

---

# Space Complexity

```
O(Cn)
```

* Storing all generated trees
* Recursion stack depth up to `O(n)`

# Takeaways

* Each number `1..n` can be treated as a **root candidate**.
* Recursively build left and right subtrees.
* Combine every possible pair of subtrees.
* Memoization avoids recomputing the same subtree ranges.
