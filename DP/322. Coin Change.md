# 💰 322. Coin Change

## 🧩 Problem Statement

You are given:
- An integer array `coins[]` representing coin denominations  
- An integer `amount`

👉 You have **infinite supply** of each coin.

Return the **minimum number of coins** needed to make up the amount.  
If it is not possible, return **-1**.

---

## 📌 Example

### Input:
coins = [1, 2, 5]  
amount = 11  

### Output:
3  

### 💡 Explanation:
11 = 5 + 5 + 1 → minimum 3 coins

---

## 🧠 Key Idea

This is an **Unbounded Knapsack (Minimization)** problem.

- You can use coins unlimited times  
- Goal: **minimize number of coins**  

---

# 🚀 Approach: 1D DP

## 💡 Idea

Let:
- `dp[i]` = minimum coins required to make amount `i`

We build the solution from smaller amounts → bigger amounts.

---

## 🧾 Code

```python
from typing import List

class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        dp = [amount + 1] * (amount + 1)
        dp[0] = 0

        for i in range(1, amount + 1):
            for coin in coins:
                if coin <= i:
                    dp[i] = min(dp[i], dp[i - coin] + 1)

        return dp[amount] if dp[amount] <= amount else -1
```

---

## 🔍 Intuition (IMPORTANT)

For each amount `i`:

👉 Try every coin as the **last coin used**

- If you pick `coin`
- Remaining = `i - coin`
- Already know best for that → `dp[i - coin]`

👉 So:
dp[i] = min(dp[i], dp[i - coin] + 1)

---

## ⚠️ Why `amount + 1`?

- It acts as **infinity**
- Worst case: cannot form amount

👉 If value remains > amount → return `-1`

---

## ⏱️ Complexity

- Time: `O(n * amount)`  
- Space: `O(amount)`  

---

# 🔥 Comparison (Important Pattern Insight)

| Problem | Goal | Operation |
|--------|------|----------|
| Coin Change II (518) | Count ways | `+` |
| Coin Change (322) | Min coins | `min()` |
| Rod Cutting | Max profit | `max()` |

👉 Same pattern → different objective

---

## 💬 Pro Tip

Whenever you see:
- “minimum / fewest” → use `min()`  
- “maximum profit” → use `max()`  
- “number of ways” → use `+`  

👉 This is the **same DP family** — only the operation changes.
