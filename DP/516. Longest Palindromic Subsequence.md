# 🔁 516. Longest Palindromic Subsequence (LPS)

## 🧩 Problem Statement

Given a string `s`, return the length of the **longest palindromic subsequence**.

👉 A subsequence does **not** need to be contiguous  
👉 Order must be maintained  

---

## 📌 Example

### Input:
s = "bbbab"

### Output:
4  

### 💡 Explanation:
One possible LPS = "bbbb"

---

## 🧠 Key Idea

👉 This problem reduces to:

```
LPS(s) = LCS(s, reverse(s))
```

So we solve using **Longest Common Subsequence (LCS)**.

---

# 🚀 Approach 1: 2D DP (Standard LCS)

## 💡 Idea

- Create DP table for LCS between `s` and `reverse(s)`
- `t[i][j]` = LCS length for first `i` chars and first `j` chars

---

## 🧾 Code

```python
class Solution:
    def longestPalindromeSubseq(self, s: str) -> int:
        s1 = s[::-1]
        n = len(s)

        t = [[0]*(n+1) for _ in range(n+1)]

        for i in range(1, n+1):
            for j in range(1, n+1):
                if s[i-1] == s1[j-1]:
                    t[i][j] = 1 + t[i-1][j-1]
                else:
                    t[i][j] = max(t[i-1][j], t[i][j-1])

        return t[n][n]
```

---

## ⏱️ Complexity

- Time: `O(n^2)`  
- Space: `O(n^2)`  

---

# ⚡ Approach 2: 1D DP (Space Optimized)

## 💡 Idea

- Reduce space from `O(n^2)` → `O(n)`
- Track diagonal using `prev`

---

## 🧾 Code

```python
class Solution:
    def longestPalindromeSubseq(self, s: str) -> int:
        s1 = s[::-1]
        n = len(s)

        dp = [0] * (n + 1)

        for i in range(1, n + 1):
            prev = 0

            for j in range(1, n + 1):
                temp = dp[j]

                if s[i-1] == s1[j-1]:
                    dp[j] = 1 + prev
                else:
                    dp[j] = max(dp[j], dp[j-1])

                prev = temp

        return dp[n]
```

---

## 🔍 Intuition (1D)

| Variable | Meaning |
|--------|--------|
| `dp[j]` | current value (t[i][j]) |
| `dp[j-1]` | left cell |
| `prev` | diagonal (t[i-1][j-1]) |

---

## ⏱️ Complexity

- Time: `O(n^2)`  
- Space: `O(n)`  

---

# 🔥 Summary

| Approach | Space | Difficulty |
|--------|------|------------|
| 2D DP | O(n²) | Easy |
| 1D DP | O(n) | Medium |

---

## ⚠️ Common Mistakes

- Using `n x n` instead of `(n+1) x (n+1)` ❌  
- Forgetting reverse string ❌  
- Wrong handling of diagonal in 1D ❌
