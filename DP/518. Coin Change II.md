# 💰 518. Coin Change II

## 🧩 Problem Statement

You are given:
- An integer `amount`
- An array `coins[]` representing different denominations

👉 You have **infinite supply** of each coin.

Return the **number of combinations** that make up the given amount.

---

## 📌 Example

### Input:
amount = 5  
coins = [1, 2, 5]

### Output:
4

### 💡 Explanation:
Possible combinations:
- 5 = 5  
- 5 = 2 + 2 + 1  
- 5 = 2 + 1 + 1 + 1  
- 5 = 1 + 1 + 1 + 1 + 1  

---

## 🧠 Key Idea

This is an **Unbounded Knapsack (Count Ways)** problem.

- You can use each coin unlimited times  
- Order **does NOT matter** (combinations, not permutations)

---

# 🚀 Approach 1: 2D DP

## 💡 Idea

Let:
- `t[i][j]` = number of ways to make amount `j` using first `i` coins

---

## 🧾 Code

```python
from typing import List

class Solution:
    def change(self, amount: int, coins: List[int]) -> int:
        n = len(coins)
        t = [[0]*(amount+1) for _ in range(n+1)]

        # Base case
        for i in range(n+1):
            t[i][0] = 1  # 1 way to make 0 (choose nothing)

        for i in range(1, n+1):
            for j in range(1, amount+1):
                if coins[i-1] <= j:
                    t[i][j] = t[i-1][j] + t[i][j - coins[i-1]]
                else:
                    t[i][j] = t[i-1][j]

        return t[n][amount]
```

---

## ⏱️ Complexity

- Time: `O(n * amount)`  
- Space: `O(n * amount)`  

---

# ⚡ Approach 2: 1D DP (Optimized)

## 💡 Idea

- `dp[x]` = number of ways to make amount `x`
- For each coin, update all possible amounts

---

## 🧾 Code

```python
from typing import List

class Solution:
    def change(self, amount: int, coins: List[int]) -> int:
        dp = [0] * (amount + 1)
        dp[0] = 1

        for coin in coins:
            for x in range(coin, amount + 1):
                dp[x] += dp[x - coin]

        return dp[amount]
```

---

## 🔍 Intuition (IMPORTANT)

For each coin:
- You try to **include it**
- Remaining amount = `x - coin`
- Ways = `dp[x - coin]`

👉 So:
dp[x] += dp[x - coin]

---

## ⚠️ Order Matters (Very Important)

### ✅ Correct (combinations)
```python
for coin in coins:
    for x in range(coin, amount+1):
```

### ❌ Wrong (permutations)
```python
for x in range(amount+1):
    for coin in coins:
```

👉 Wrong order will count duplicates like:
- (2+1+1+1)  
- (1+2+1+1)  
- (1+1+2+1)  

---

## ⏱️ Complexity

- Time: `O(n * amount)`  
- Space: `O(amount)`  

---

# 🔥 Summary

- This is **Unbounded Knapsack (count ways)**
- Use:
  - `+` for counting ways  
  - `max()` for optimization problems  
- Loop order decides:
  - combinations ✅  
  - permutations ❌  

---

## 💬 Pro Tip

If question says:
- “number of ways” → use `+`
- “maximum profit/value” → use `max`

👉 Same pattern, different goal.
