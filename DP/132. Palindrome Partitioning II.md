# 🔪 132. Palindrome Partitioning II

## 🧩 Problem Statement

Given a string `s`, partition it such that **every substring is a palindrome**.

👉 Return the **minimum number of cuts** needed.

---

## 📌 Example

### Input:
s = "aab"

### Output:
1  

### 💡 Explanation:
Partition: ["aa", "b"] → 1 cut

---

## 🧠 Key Idea

We need to:
👉 Split the string so that **all parts are palindromes**  
👉 Minimize number of cuts  

---

# 🚀 Approach 1: Recursion (Brute Force)

## 💡 Idea

Try all partitions:
```
(i → k) + (k+1 → j)
```

---

## 🧾 Code

```python
class Solution:
    def minCut(self, s: str) -> int:
        return self.solve(s, 0, len(s)-1)

    def isPalindrome(self, s, i, j):
        return s[i:j+1] == s[i:j+1][::-1]

    def solve(self, s, i, j):
        if i >= j or self.isPalindrome(s, i, j):
            return 0

        mn = float('inf')

        for k in range(i, j):
            temp = (
                self.solve(s, i, k)
                + self.solve(s, k+1, j)
                + 1
            )
            mn = min(mn, temp)

        return mn
```

---

## ⏱️ Complexity

- Time: `O(2^n)` ❌ (TLE)  
- Space: recursion stack  

---

# ⚡ Approach 2: Memoization (Top-Down DP)

## 💡 Idea

Store results of `(i, j)` to avoid recomputation.

---

## 🧾 Code

```python
class Solution:
    def minCut(self, s: str) -> int:
        n = len(s)
        t = [[-1]*n for _ in range(n)]
        return self.solve(s, 0, n-1, t)

    def isPalindrome(self, s, i, j):
        while i < j:
            if s[i] != s[j]:
                return False
            i += 1
            j -= 1
        return True

    def solve(self, s, i, j, t):
        if i >= j:
            return 0

        if t[i][j] != -1:
            return t[i][j]

        if self.isPalindrome(s, i, j):
            t[i][j] = 0
            return 0

        mn = float('inf')

        for k in range(i, j):
            if t[i][k] != -1:
                left = t[i][k]
            else:
                left = self.solve(s, i, k, t)

            if t[k+1][j] != -1:
                right = t[k+1][j]
            else:
                right = self.solve(s, k+1, j, t)

            mn = min(mn, left + right + 1)

        t[i][j] = mn
        return mn
```

---

## ⏱️ Complexity

- Time: `O(n^3)` ⚠️  
- Space: `O(n^2)`  

👉 Still TLE for large inputs

---

# 🚀 Approach 3: Optimal (O(n²) DP)

## 💡 Idea

👉 Switch from partition DP → prefix DP

### Step 1:
Precompute palindrome table:
```
isPal[i][j] = True if s[i:j] is palindrome
```

### Step 2:
```
dp[i] = minimum cuts for substring s[0:i]
```

---

## 🧾 Code

```python
class Solution:
    def minCut(self, s: str) -> int:
        n = len(s)

        # Step 1: Palindrome table
        isPal = [[False]*n for _ in range(n)]

        for i in range(n):
            for j in range(i+1):
                if s[i] == s[j] and (i - j <= 2 or isPal[j+1][i-1]):
                    isPal[j][i] = True

        # Step 2: DP for min cuts
        dp = [0]*n

        for i in range(n):
            if isPal[0][i]:
                dp[i] = 0
            else:
                dp[i] = float('inf')
                for j in range(i):
                    if isPal[j+1][i]:
                        dp[i] = min(dp[i], dp[j] + 1)

        return dp[n-1]
```

---

## ⏱️ Complexity

- Time: `O(n^2)` ✅  
- Space: `O(n^2)`  

---

# 🔥 Summary

| Approach | Time | Space | Status |
|--------|------|------|--------|
| Recursion | Exponential ❌ | Stack | TLE |
| Memoization | O(n³) ⚠️ | O(n²) | Still slow |
| Optimal DP | O(n²) ✅ | O(n²) | Best |

---

# 🧠 Key Insight

👉 Not all partition problems need MCM-style DP  

👉 This problem is better solved as:
```
Prefix DP + Palindrome Preprocessing
```
