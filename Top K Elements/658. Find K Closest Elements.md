# 🔎 658. Find K Closest Elements

## 🧩 Problem

You are given a **sorted array** `arr`, and two integers `k` and `x`.

🎯 **Goal:**
Return the **k closest elements** to `x` in the array.

### 📌 Rules for closeness:

An element `a` is closer than `b` if:

* `|a - x| < |b - x|`
* OR if equal → **smaller value wins**

---

## 💡 Key Insight

👉 Since the array is **already sorted**, the answer will always be:

> 🔑 A **continuous subarray of size k**

So instead of picking elements one by one:

```text
❌ Pick k elements individually
```

We do:

```text
✅ Find the best window of size k
```

---

## 🧠 Strategy (Binary Search on Window)

We search for the **starting index** of the window.

### Possible starting range:

```python
left = 0
right = len(arr) - k
```

Why?

* Window size = `k`
* Last valid start = `n - k`

---

## 🔁 Binary Search Logic

At each step:

```python
mid = (left + right) // 2
```

We compare:

```python
x - arr[mid]   vs   arr[mid + k] - x
```

---

## ⚖️ Decision Making

### Case 1️⃣

```python
x - arr[mid] > arr[mid + k] - x
```

👉 Left side is farther → shift right

```python
left = mid + 1
```

---

### Case 2️⃣

```python
x - arr[mid] <= arr[mid + k] - x
```

👉 Right side is farther (or equal) → shift left

```python
right = mid
```

---

## 💻 Python Implementation

```python
class Solution:
    def findClosestElements(self, arr: List[int], k: int, x: int) -> List[int]:
        left = 0
        right = len(arr) - k

        while left < right:
            mid = (left + right) // 2

            if x - arr[mid] > arr[mid + k] - x:
                left = mid + 1
            else:
                right = mid

        return arr[left:left + k]
```
---

## ⏱ Complexity Analysis

| Type     | Complexity               |
| -------- | ------------------------ |
| ⏱ Time   | **O(log(n - k)) + O(k)** |
| 📦 Space | **O(1)**                 |

---

## ⚠️ Edge Cases

✔ `x` smaller than all elements
✔ `x` larger than all elements
✔ `k = len(arr)`
✔ Duplicate values
