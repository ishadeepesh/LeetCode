# 🔎 1481. Least Number of Unique Integers after K Removals

## 🧩 Problem

You are given an array `arr` and an integer `k`.

🎯 **Goal:**
Remove exactly `k` elements such that the number of **unique integers remaining is minimized**.

---

## 📌 Example

**Input**

```python
arr = [5,5,4]
k = 1
```

**Output**

```text
1
```

**Explanation**

Remove `4` → remaining `[5,5]` → only **1 unique number**

---

# 💡 Key Insight (Greedy)

> 🔑 Always remove numbers with **smallest frequency first**

Why?

* Removing a number with frequency `1` reduces unique count immediately
* Removing a number with frequency `5` is expensive

---

# 🧠 Approach 1: Min Heap (Greedy)

### Idea

1. Count frequency of each number
2. Push `(frequency, number)` into a **min heap**
3. Keep removing smallest frequency elements until `k` is exhausted

---

## 💻 Code

```python
import heapq
from collections import Counter

class Solution:
    def findLeastNumOfUniqueInts(self, arr, k):
        count = Counter(arr)

        heap = []
        for num, freq in count.items():
            heap.append((freq, num))

        heapq.heapify(heap)

        while k > 0:
            freq, num = heapq.heappop(heap)

            if freq <= k:
                k -= freq
            else:
                return len(heap) + 1

        return len(heap)
```

---

## ⏱ Complexity

| Type     | Complexity     |
| -------- | -------------- |
| ⏱ Time   | **O(n log n)** |
| 📦 Space | **O(n)**       |

---

## ✅ Pros

✔ Easy to understand
✔ Standard greedy approach
✔ Interview-safe

---

# 🧠 Approach 2: Bucket Sort (Optimal)

### Idea

Instead of heap:

* Count how many numbers have a certain frequency
* Remove from smallest frequency → largest

---

## 💻 Code

```python
from collections import Counter

class Solution:
    def findLeastNumOfUniqueInts(self, arr, k):
        count = Counter(arr)

        freq = [0] * (len(arr) + 1)

        for c in count.values():
            freq[c] += 1

        unique = len(count)

        for i in range(1, len(freq)):
            if k >= i * freq[i]:
                k -= i * freq[i]
                unique -= freq[i]
            else:
                unique -= k // i
                break

        return unique
```

---

## ⏱ Complexity

| Type     | Complexity |
| -------- | ---------- |
| ⏱ Time   | **O(n)** ✅ |
| 📦 Space | **O(n)**   |

---

## ✅ Pros

✔ Optimal time complexity
✔ No heap required
✔ Efficient for large inputs
