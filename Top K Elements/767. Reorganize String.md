# 🔎 767. Reorganize String

## 🧩 Problem

Given a string `s`, rearrange the characters such that **no two adjacent characters are the same**.

🎯 **Goal:**

* Return any valid rearranged string
* If not possible → return `""`

---

## 📌 Example

**Input**

```python
s = "aab"
```

**Output**

```text
"aba"
```

---

# 💡 Key Insight

> 🔑 Always pick the **most frequent character**, but avoid placing the same character twice in a row.

---

# 🧠 Strategy (Greedy + Max Heap)

1️⃣ Count frequency of each character
2️⃣ Store them in a **max heap** (based on frequency)
3️⃣ Always pick the **most frequent character**
4️⃣ Keep track of the **previous character** to avoid repetition
5️⃣ Reinsert previous character back into heap after one step

---

# 💻 Python Implementation

```python
import heapq
from collections import Counter

class Solution:
    def reorganizeString(self, s: str) -> str:
        count = Counter(s)

        # max heap (negative count for max behavior)
        maxHeap = [[-cnt, char] for char, cnt in count.items()]
        heapq.heapify(maxHeap)

        prev = None
        res = ""

        while maxHeap or prev:

            # ❌ No valid character to place
            if prev and not maxHeap:
                return ""

            cnt, char = heapq.heappop(maxHeap)

            # add current character
            res += char
            cnt += 1  # reduce count (since negative)

            # push previous character back
            if prev:
                heapq.heappush(maxHeap, prev)
                prev = None

            # store current if still remaining
            if cnt != 0:
                prev = [cnt, char]

        return res
```

# 🔍 Example Walkthrough

```python
s = "aab"
```

### Step-by-step:

```text
Heap: [(-2,'a'), (-1,'b')]

Pick 'a' → res = "a"
Store prev = ('a', remaining)

Pick 'b' → res = "ab"
Push 'a' back

Pick 'a' → res = "aba"
```

---

# ⚠️ Edge Case

```python
s = "aaab"
```

```text
Impossible → return ""
```

Why?

```text
One character appears too many times → cannot avoid adjacency
```

---

# 🧠 Validity Condition (Important)

A valid rearrangement exists only if:

```text
max_frequency ≤ (n + 1) // 2
```

---

# ⏱ Complexity

| Type     | Complexity              |
| -------- | ----------------------- |
| ⏱ Time   | **O(n log k)** (≈ O(n)) |
| 📦 Space | **O(k)**                |

---

# ⚖️ Why this works

* Always placing the **most frequent available character**
* Delaying reuse using `prev`
* Ensures no two adjacent characters are same

---

# 🔑 Final Takeaway

```text
Use a max heap + delay mechanism to avoid placing same characters consecutively
```
