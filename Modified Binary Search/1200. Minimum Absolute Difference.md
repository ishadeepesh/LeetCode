# 📉 1200. Minimum Absolute Difference

## 📌 Problem Statement

Given an array of **distinct integers** `arr`, find all pairs of elements with the **minimum absolute difference** between any two elements.

Return a list of pairs in **ascending order**. Each pair `[a, b]` must satisfy:

* `a` and `b` are elements from `arr`
* `a < b`
* `b - a` equals the **minimum absolute difference** of any two elements in the array

---

## 🧾 Example

### Example 1

**Input**

```text
arr = [4,2,1,3]
```

**Output**

```text
[[1,2],[2,3],[3,4]]
```

### Explanation

After sorting the array:

```text
[1,2,3,4]
```

Differences between adjacent elements:

```text
2-1 = 1
3-2 = 1
4-3 = 1
```

The **minimum absolute difference is 1**, so we return all pairs with difference **1**.

---

# 💡 Approach

The key observation is:

👉 The **minimum difference will always occur between adjacent elements after sorting**.

### Steps

1️⃣ **Sort the array**

Sorting ensures that the smallest differences occur between **neighboring elements**.

2️⃣ **Traverse the array once**

For each pair of adjacent elements:

```text
difference = arr[i] - arr[i-1]
```

3️⃣ **Track the minimum difference**

* If a **smaller difference** is found:

  * Update `minDiff`
  * Reset the result list

* If the difference **equals the minimum difference**:

  * Append the pair to the result list

---

# 🧠 Key Insight

Without sorting, checking every pair would take:

```text
O(n²)
```

But sorting lets us compare only **adjacent elements**, reducing the work significantly.

---

# 🧩 Python Implementation

```python
from typing import List

class Solution:
    def minimumAbsDifference(self, arr: List[int]) -> List[List[int]]:
        arr.sort()
        minDiff = float('inf')
        res = []

        for i in range(1, len(arr)):
            d = arr[i] - arr[i-1]

            if d < minDiff:
                minDiff = d
                res = [[arr[i-1], arr[i]]]

            elif d == minDiff:
                res.append([arr[i-1], arr[i]])

        return res
```

---

# ⏱️ Time Complexity

```text
O(n log n)
```

* Sorting the array → `O(n log n)`
* Single traversal → `O(n)`

Total complexity is dominated by sorting.

---

# 💾 Space Complexity

```text
O(1)
```

Only a result list and a few variables are used.

---

# 🏁 Summary

✨ Sort the array first
✨ Compare only **adjacent elements**
✨ Track the **minimum difference** during traversal
✨ Reset results when a smaller difference is found

This problem demonstrates a common pattern:

> **Sorting simplifies pair comparison problems.**
