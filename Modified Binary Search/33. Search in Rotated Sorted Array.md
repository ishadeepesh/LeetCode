# 🔎 33. Search in Rotated Sorted Array

## 🧩 Problem

You are given an integer array `nums` that was originally **sorted in ascending order** with **distinct values**.

However, before reaching your function, the array might have been **rotated** at some unknown index.

Example:

Original array

```python
[0,1,2,4,5,6,7]
```

After rotation (k = 3)

```python
[4,5,6,7,0,1,2]
```

🎯 **Goal:**
Given `nums` and a `target`, return the **index of the target** if it exists.
If not found, return **-1**.

⚡ **Constraint:**
Your solution must run in **O(log n)** time.

---

## 💡 Key Insight

Even though the array is rotated, **one half of the array is always sorted**.

Example:

```
[4,5,6,7 | 0,1,2]
```

* Left half → sorted 📈
* Right half → sorted 📈

So during **binary search**, we:

1. Find the `mid`
2. Detect **which half is sorted**
3. Check if the **target lies in that half**
4. Continue searching there

This keeps the runtime **logarithmic** 🚀.

---

## 🧠 Strategy

During each iteration:

1️⃣ Calculate the middle index
2️⃣ Check if we found the target
3️⃣ Determine which side is sorted
4️⃣ Decide whether to search **left** or **right**

---

## 🧑‍💻 Python Implementation

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        lo, hi = 0, len(nums) - 1

        while lo <= hi:
            mid = (lo + hi) // 2

            # 🎯 Target found
            if nums[mid] == target:
                return mid

            # 📈 Left half is sorted
            if nums[0] <= nums[mid]:
                if nums[lo] <= target <= nums[mid]:
                    hi = mid - 1
                else:
                    lo = mid + 1

            # 📈 Right half is sorted
            else:
                if nums[mid] <= target <= nums[hi]:
                    lo = mid + 1
                else:
                    hi = mid - 1

        return -1
```

---

## 📌 Example

**Input**

```python
nums = [4,5,6,7,0,1,2]
target = 0
```

**Output**

```
4
```

📍 Because `0` is located at **index 4**.

---

## ⏱ Complexity Analysis

| Type     | Complexity   |
| -------- | ------------ |
| ⏱ Time   | **O(log n)** |
| 📦 Space | **O(1)**     |

Binary search halves the search space every iteration.

---

## ⚠️ Edge Cases

✔ Target not present → return `-1`
✔ Array not rotated
✔ Array rotated by `n-1` positions
✔ Single element array

---

## 🏁 Final Takeaway

The trick is simple but powerful:

> 🔑 **At every step, one half of the array is guaranteed to be sorted.**

Use that information to eliminate half the search space — just like binary search.

🚀 Fast, elegant, and interview-favorite problem!
