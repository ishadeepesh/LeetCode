# 📍 34. Find First and Last Position of Element in Sorted Array

## 📌 Problem Statement

Given an array of integers `nums` sorted in **non-decreasing order**, find the **starting and ending position** of a given `target` value.

If the `target` is **not found**, return:

```text
[-1, -1]
```

⚠️ The algorithm must run in **O(log n)** time complexity.

---

## 🧾 Example

### Example 1

**Input**

```text
nums = [5,7,7,8,8,10]
target = 8
```

**Output**

```text
[3,4]
```

### Explanation

The value `8` appears in the array at indices **3** and **4**.
So the starting index is **3** and the ending index is **4**.

---

### Example 2

**Input**

```text
nums = [5,7,7,8,8,10]
target = 6
```

**Output**

```text
[-1,-1]
```

### Explanation

`6` does not exist in the array.

---

# 💡 Approach

Since the array is **sorted**, we can use **Binary Search** to find the target efficiently.

However, we must find:

* The **first occurrence** of the target
* The **last occurrence** of the target

To achieve this, we perform **Binary Search twice**:

1️⃣ **Left-biased Binary Search** → finds the **first position**
2️⃣ **Right-biased Binary Search** → finds the **last position**

---

## 🔎 Key Idea

When we find the target:

* If searching for the **left boundary**, continue searching on the **left side**.
* If searching for the **right boundary**, continue searching on the **right side**.

This ensures we locate the **extreme positions** of the target.

---

# 🧠 Algorithm Steps

### Step 1 — Find the Leftmost Index

* Perform Binary Search
* When `nums[mid] == target`, store the index
* Move **left** to check if an earlier occurrence exists

---

### Step 2 — Find the Rightmost Index

* Perform Binary Search again
* When `nums[mid] == target`, store the index
* Move **right** to check for later occurrences

---

# 🧩 Python Implementation

```python
from typing import List

class Solution:
    def searchRange(self, nums: List[int], target: int) -> List[int]:
        left = self.binSearch(nums, target, True)
        right = self.binSearch(nums, target, False)
        return [left, right]

    def binSearch(self, nums, target, leftBias):
        lo, hi, i = 0, len(nums) - 1, -1

        while lo <= hi:
            mid = (lo + hi) // 2

            if target > nums[mid]:
                lo = mid + 1
            elif target < nums[mid]:
                hi = mid - 1
            else:
                i = mid
                if leftBias:
                    hi = mid - 1
                else:
                    lo = mid + 1

        return i
```

---

# ⏱️ Time Complexity

```text
O(log n)
```

We perform **Binary Search twice**, but each search still runs in logarithmic time.

---

# 💾 Space Complexity

```text
O(1)
```

Only constant extra space is used.

---

# 🏁 Summary

✨ Uses **Binary Search** on a sorted array
✨ Runs in **O(log n)** time
✨ Performs **two searches** to find left and right boundaries
✨ Returns `[-1, -1]` if the target is not present

This is a classic **boundary binary search pattern**, often used when duplicates exist in a sorted array.
