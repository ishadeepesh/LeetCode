# 🔤 744. Find Smallest Letter Greater Than Target

## 📌 Problem Statement

You are given a **sorted array of characters** `letters` (sorted in **non-decreasing order**) and a character `target`.

Your task is to return the **smallest character in the array that is lexicographically greater than the target**.

If such a character **does not exist**, return the **first character in the array** (because the letters wrap around).

⚠️ There are **at least two different characters** in the array.

---

## 🧾 Example

### Example 1

**Input**

```text id="c8n8w4"
letters = ["c","f","j"]
target = "a"
```

**Output**

```text id="c8qv4t"
"c"
```

### Explanation

The smallest character that is lexicographically greater than `'a'` is **`'c'`**.

---

### Example 2

**Input**

```text id="t3a4gr"
letters = ["c","f","j"]
target = "c"
```

**Output**

```text id="czzv3t"
"f"
```

### Explanation

The next greater character after `'c'` is **`'f'`**.

---

### Example 3

**Input**

```text id="yxgpt2"
letters = ["c","f","j"]
target = "j"
```

**Output**

```text id="u40g7r"
"c"
```

### Explanation

There is **no character greater than `'j'`**, so we return the **first character** in the array due to wrap-around.

---

# 💡 Approach

This problem can be solved efficiently using **Binary Search** because the array is **sorted**.

We want to find the **smallest element strictly greater than `target`**.

---

## 🧠 Key Idea

If the **last element** in the array is **less than or equal to the target**, then **no element is greater**, so we return the **first element**.

```text id="xk9l0y"
letters[n-1] <= target
```

Otherwise, perform **Binary Search** to find the first letter greater than the target.

---

## 🔎 Binary Search Steps

1. Initialize two pointers:

```text id="0s2p5x"
left = 0
right = n - 1
```

2. While `left <= right`:

* Compute the middle index

```text id="u9l1u5"
mid = (left + right) // 2
```

* If `letters[mid] > target`

  * Move **left side of search space**

```text id="qbb0p3"
right = mid - 1
```

* Otherwise

  * Move **right side**

```text id="9ypxhc"
left = mid + 1
```

3. After the loop finishes, `left` will point to the **smallest character greater than target**.

---

# 🧩 Python Implementation

```python id="d76h8u"
from typing import List

class Solution:
    def nextGreatestLetter(self, letters: List[str], target: str) -> str:
        n = len(letters)

        if letters[n-1] <= target:
            return letters[0]

        left = 0
        right = n - 1

        while left <= right:
            mid = (left + right) // 2

            if letters[mid] > target:
                right = mid - 1
            else:
                left = mid + 1

        return letters[left]
```

---

# ⏱️ Time Complexity

```text id="81m2q7"
O(log n)
```

Binary search reduces the search space by half each iteration.

---

# 💾 Space Complexity

```text id="a8l5ib"
O(1)
```

Only constant extra space is used.
