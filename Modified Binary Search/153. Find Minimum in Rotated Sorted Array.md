# 🔎 153. Find Minimum in Rotated Sorted Array

## 🧩 Problem

An array sorted in **ascending order** is rotated between **1 and n times**.

Example:

Original array

```python
[0,1,2,4,5,6,7]
```

Possible rotations

```python
[4,5,6,7,0,1,2]
[0,1,2,4,5,6,7]
```

🎯 **Goal**

Given a rotated sorted array `nums` with **unique elements**, return the **minimum element**.

⚡ Your algorithm must run in **O(log n)** time.

---

# 💡 Key Insight

In a rotated sorted array:

```
[4,5,6,7 | 0,1,2]
```

The **minimum element is the pivot point** where the rotation happened.

Instead of scanning the array (`O(n)`), we can use **binary search**.

---

# 🧠 Strategy

We compare `nums[mid]` with `nums[right]`.

### Case 1

```
nums[mid] < nums[right]
```

Example:

```
[4,5,6,7 | 0,1,2]
       mid     right
```

This means the **minimum lies in the left half (including mid)**.

```
right = mid
```

---

### Case 2

```
nums[mid] > nums[right]
```

Example:

```
[4,5,6 | 7,0,1,2]
      mid      right
```

This means the **minimum lies in the right half**.

```
left = mid + 1
```

---

# 💻 Python Implementation

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:

        left, right = 0, len(nums) - 1

        while left < right:

            mid = left + (right - left) // 2

            # minimum lies in left part (including mid)
            if nums[mid] < nums[right]:
                right = mid

            # minimum lies in right part
            else:
                left = mid + 1

        return nums[left]
```

---

# 📌 Example

**Input**

```python
nums = [3,4,5,1,2]
```

**Output**

```
1
```

Explanation:

Original array:

```
[1,2,3,4,5]
```

After rotation:

```
[3,4,5,1,2]
       ↑
     minimum
```

---

# ⏱ Complexity Analysis

| Type     | Complexity   |
| -------- | ------------ |
| ⏱ Time   | **O(log n)** |
| 📦 Space | **O(1)**     |

Binary search reduces the search space by **half each iteration**.

---

# ⚠️ Edge Cases

✔ Array not rotated → `[1,2,3,4]`
✔ Array rotated once → `[4,1,2,3]`
✔ Array with only one element

---

# 🔑 Final Takeaway

The trick is recognizing that:

> In a rotated sorted array, **one half is always sorted**.

By comparing `nums[mid]` with `nums[right]`, we can **eliminate half of the array each step**.

🚀 This gives a clean **O(log n)** binary search solution.
